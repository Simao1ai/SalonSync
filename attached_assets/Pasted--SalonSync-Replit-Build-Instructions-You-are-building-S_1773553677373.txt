# SalonSync — Replit Build Instructions

You are building **SalonSync**, a production-grade AI-powered SaaS platform for hair salons. Read this entire file before writing a single line of code. Follow every instruction exactly.

-----

## What You Are Building

A multi-tenant SaaS web application for hair salons with three user roles (Admin, Staff, Client), AI-powered features, appointment scheduling, Stripe payments, and cancellation fee enforcement.

**This is a real product. Build it to production quality — not a demo.**

-----

## Design Direction — CRITICAL

SalonSync must look like a **premium, modern SaaS product**. Think Linear, Vercel dashboard, or Stripe’s UI — clean, confident, and professional.

**Visual Identity:**

- **Color palette:** Deep slate/charcoal base (`#0F172A`) with a warm rose-gold accent (`#C9956A`) for a luxury salon feel
- **Typography:** Use `Playfair Display` for headings (elegant, salon-appropriate) and `DM Sans` for body text (modern, readable)
- **Style:** Refined minimalism — generous white space, subtle shadows, smooth transitions
- **Components:** Rounded cards with subtle borders, clean data tables, smooth modals
- **Avoid:** Purple gradients, generic blue buttons, clipart icons, generic AI aesthetics

**Import these fonts in layout.tsx:**

```
Playfair Display (weights: 400, 600, 700)
DM Sans (weights: 400, 500, 600)
```

### Admin Dashboard (`/admin/dashboard`)

**Design:** Left sidebar (dark slate `#0F172A`) + main content area (light gray `#F8FAFC`).

**Sidebar contains:**

- SalonSync logo at top
- Navigation links: Dashboard, Appointments, Staff, Clients, Services, Analytics, Settings
- Active state: rose-gold left border + light background
- User name + role badge at bottom + logout button

**Main dashboard shows 6 stat cards:**

1. Today’s Revenue (`$`)
1. Appointments Today (count)
1. Cancellation Rate (%)

**Below stats:**

**Shows:**

### Booking Flow (`/client/book`)

Build a multi-step booking form:

**Step 1 — Select Service:**

- Grid of service cards (name, price, duration, category badge)
- HIGH_VALUE services show a “Deposit Required” badge in amber
- Search/filter by category

**Step 2 — Select Stylist:**

- Show only stylists available for selected service
- Stylist cards: photo placeholder, name, specialties, star rating

**Step 3 — Select Date & Time:**

- Calendar component showing available dates

**Step 4 — Confirm & Pay:**

- Appointment summary

## API Routes to Build

### `POST /api/appointments`

- Validate all fields
- Check stylist availability (no overlapping appointments)
- Calculate total price from services
- Detect if HIGH_VALUE
- Create Appointment + AppointmentService records
- Trigger AI risk scoring (async, don’t block response)
- Schedule reminders (48hr and 24hr)
- Return created appointment

### `PATCH /api/appointments/[id]/cancel`

- Enforce cancellation policy
- Charge or refund via Stripe
- Update appointment status
- Send cancellation confirmation email via Resend

### `GET /api/appointments`

- Admin: all appointments for their location(s)
- Staff: only their own appointments
- Client: only their own appointments
- Support query params: `?date=`, `?status=`, `?staffId=`

### `POST /api/ai/chat`

- Accept `{ messages, locationId }` body
- Fetch location’s services for context
- Call `chatWithReceptionist()` from claude.ts
- Return AI response

### `POST /api/webhooks/stripe`

- Handle `payment_intent.succeeded`
- Handle `payment_intent.payment_failed`
- Update PaymentStatus on Appointment accordingly

## Pricing Model (for Settings page)

- Base fee: **$20/month per location**
- Per seat fee: **$10/month** (owners + all staff + secretaries)
- Example: 1 location, 5 seats = **$70/month**
- Multi-location: base fee charged per location

-----

## Key Business Rules — Never Break These

1. HIGH_VALUE appointments (highlights, extensions) **always** require payment at booking
1. Cancellation window default = **48 hours** — charge fee if cancelled after this
1. No-show = **full charge** applied automatically
1. Client must check a **policy acknowledgment checkbox** before confirming any booking
1. Admin can see **all locations** — staff and clients see **only their location**
1. Stripe amounts are always in **cents** (multiply dollars by 100)
1. All dates stored in **UTC**, displayed in location timezone
1. Never expose `SUPABASE_SERVICE_ROLE_KEY` or `STRIPE_SECRET_KEY` to the client
